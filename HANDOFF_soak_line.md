# 酸洗浸泡產線 — 工程交接文件 (Handoff)

> 給接手工程師 / Agent：本文件描述「酸洗浸泡產線模擬 Soak Line」原型的**業務規則、資料模型、狀態機、整合契約**。原型為**純前端、含模擬資料**的可運行 demo，目的在凍結整條產線的 UX 與判定邏輯；生產化時把模擬資料層換成真實的 **RFID 讀取**與**上位系統 (Host/MES)** 串接，並補上後端與紀錄持久化。

- **狀態**：原型完成、邏輯凍結；待生產化串接。
- **範圍**：涵蓋整條產線 —— 簽入 Signin、進料檢驗 IQC、酸洗浸泡 Tank、超純水潤洗 PW、最終潤洗 HPW，以及異常處理、現場螢幕、監控站。
- **時間數值皆為示意**（要求秒數、寬限、demo 加速倍率），正式值須向流程 Runcard owner 取得。
- **語言**：UI 為繁中／英雙語（可切換）；程式識別字、API 欄位用英文。

---

## 1. 專案目的

工廠零件酸洗浸泡時，零件貼有 RFID tag、依製程逐站流動並置入酸桶浸泡。本系統以 RFID 全程追蹤**每件零件的位置與浸泡時間**，依零件規格判定「達標 / 超時 / 異常」，在異常時即時警示現場人員，並於離槽時留下可稽核紀錄。

## 3. 核心業務規則（**不可變動**）

以下為需求方要求並已實作於原型的業務規則，是本系統的判定核心。

### 規則 1 — 單向流程，不可回頭 / 不可跳站
- 零件必須依序通過：**Signin → IQC → 酸洗浸泡 (Tank) → HPW（結束）**。
- **不得倒退**（如 IQC 退回 Signin）、**不得跳站**（如未 IQC 直接進 Tank）。
- **例外**：酸洗浸泡（Tank，清洗區）與**移出區 (Removal)** 之間可自由互換（見規則 6）。
- 進 IQC 前必須已 Signin；進 Tank 前必須已完成 Signin + IQC。

### 規則 2 — 一桶一酸（酸種為酸桶層級屬性）
- 同一酸桶 (Tank) **只裝一種液體** —— 某一種酸，或超純水 (PW)。
- 酸代號**不是**零件可選欄位，而是**酸桶的設定**。
- 衍生防呆：零件規格酸種 ≠ 本桶酸種 → **錯槽 (WRONG)**，禁止進槽。

### 規則 3 — 酸種配對（含 PW 前置條件）
- 進酸桶（酸）：零件**規格酸種**必須與桶內酸種**一致**，否則錯槽、禁止進槽。
- 進超純水桶 (PW)：零件必須**已先泡過酸**（曾在相符酸桶完成浸泡）才可進入；未泡酸者禁止直接進 PW，確保潤洗發生在正確順序之後。

### 規則 4 — 浸泡參數依 PN/SN 帶回（來自上位系統）
- 酸種與要求浸泡時間**不寫死**：RFID 讀到零件 PN/SN 後向上位系統查得 `{規格酸種, 要求時間}`。
- **同一桶內不同零件可有不同要求時間**（要求是 per-PN，不是 per-tank）。

### 規則 5 — 現場到「秒」，判定 / 紀錄到「分」（1 分鐘寬限）
- **現場計時**：零件在槽內即時**往上累計**已浸泡時間，顯示到**秒** (`M:SS`)；達標後**不停**，持續累計。
- **達標 / 可取出**：已浸泡時間 ≥ 要求時間 → 「達標・可取出」（綠）。
- **1 分鐘寬限**：達標後 1 分鐘內仍視為達標可取出，**不算超時**（避免差幾秒被誤判 NG）。
- **超時 (OVER)**：超過要求時間 **1 分鐘以上**才判定超時（紅），並記錄超時多久。
- **最終判定以「分」認定**：達標 PASS / 超時 OVER / 異常 NG。

### 規則 6 — 異常處理（RFID 在槽 / 離槽 + 未偵測 Tag）
- **未達標即被取出**（浸泡中被拿走）→ 判定「**異常**」。
  - 系統**保留該零件已浸泡的時間**；日後放回任一相符酸桶時可**接續計時**，不需從頭重泡。
  - 異常標記**跟著零件**，直到零件進 HPW 完成或被重置。
- **未偵測到 Tag / 請放回**：零件在浸泡中被取出、**尚未放回任何酸桶**時，其來源酸桶對應的**現場螢幕顯示紅色警示「未偵測到 Tag・請放回」**；放回（接續計時）或完成流程後解除。
- **移出區**：被取出的零件可暫置於此，代表目前不在任何酸桶中、等待放回或處置。
- **進 HPW = 流程結束**：解除該零件所有製程標記（Signin / IQC / 泡酸紀錄 / 異常 / 保留時間）。

## 4. 資料模型

### 4.1 流程卡 Runcard（執行期）
- `id`：流程卡號（RC-…）。
- `pn` / `sn`：料號 / 序號（RFID 讀回）。
- `requiredAcid`：規格酸種（由 PN 帶回；規則 3/4）。
- `soakSec`：要求浸泡時間（由 PN 帶回；規則 4）。
- `signin` / `iqc`：是否已簽入 / 已檢驗（規則 1）。
- `acidSoaked`：是否已泡過酸（PW 前置條件；規則 3）。
- `abnormal`：是否帶異常（規則 6）。
- `elapsedSec`：異常取出時保留的已計時秒（供接續計時；規則 6）。
- `pulledFromTank`：最近一次被異常取出的來源槽（驅動「請放回」警示；規則 6）。
- `location`：目前位置（pool / signin / iqc / <tankId> / removed / hpw）。

### 4.2 酸桶 Tank（規則 2）
- `id` / `label`：酸桶識別與名稱（Tank A/B…）。
- `benchId`：所在 Soak Bench（可調度移動）。
- `acid`：桶內唯一液體（某酸 或 PW）；生產時來自酸桶設定 / MES。
- `runcardId`：槽內零件（無則空）。
- `totalSec` / `elapsedSec`：要求時間 / 已計時（規則 5）。
- `status`：`running`（未達標）/ `done`（達標寬限內）/ `over`（超時）/ `idle`（空）。
- `abnormal`：槽內零件是否帶異常提示。

### 4.3 出槽紀錄 Record（規則 5，稽核用）
- `rc / pn / sn / acid`：零件與所泡液體。
- `reqSec / actualSec / overtimeSec`：要求 / 實際 / 超時（秒）。
- `result`：`pass` 達標 / `over` 超時 / `abnormal` 異常 / `rinse` PW 完成。
- `enter / out`：**入槽 / 出槽時間，記錄到「分」**（HH:MM，不記錄到秒）。
- 註：入槽與出槽事件另會寫入操作紀錄 (Log)，時間同樣到「分」。

## 5. 狀態機與判定邏輯

### 5.1 流程站序（規則 1）
`pool → signin → iqc → tank(清洗區) → hpw`；`tank ↔ removed` 可互換。
每次移動先檢查站序（不可回頭 / 跳站），再檢查該站專屬規則（酸種配對、占用等）。

### 5.2 進槽判定（規則 2/3）
| 條件 | 結果 |
|---|---|
| 未 Signin+IQC | 禁止進槽 |
| 槽已被占用 | 禁止 |
| 槽未倒入液體 | 禁止 |
| 槽為酸且規格酸種 ≠ 本桶 | 錯槽，禁止 |
| 槽為 PW 且零件未泡過酸 | 禁止 |
| 以上皆通過 | 進槽並開始計時（接續已保留時間） |

### 5.3 槽內即時狀態（規則 5，秒）
令 `elapsed` = 已計時、`total` = 要求、`GRACE` = 60 秒。
| 狀態 | 條件 | 顏色 |
|---|---|---|
| `running` 作業中 | `elapsed < total` | 一般 |
| `done` 達標可取出 | `total ≤ elapsed < total+GRACE` | 綠 |
| `over` 超時 | `elapsed ≥ total+GRACE` | 紅 |

**疊加提示**：`abnormal`（曾未達標被取出）持續顯示紅色異常；`pulled`（被取出未放回）於來源槽現場螢幕顯示「未偵測到 Tag・請放回」。

### 5.4 離槽最終判定（規則 5，分）
| result | 條件 | 標籤 |
|---|---|---|
| `abnormal` | 未達標被取出，或帶異常歷史 | 異常 NG（紅） |
| `over` | 超過要求時間 1 分鐘以上 | 超時 OVER（紅） |
| `pass` | 達標且在寬限內 | 達標 PASS（綠） |
| `rinse` | PW 潤洗完成 | PW 完成 |

### 5.5 全線彙總警示
任何槽存在 `over` / `abnormal`，或任何零件「被取出未放回」，於監控站頂部橫幅彙總件數。

## 6. 整合點：目前 mock vs 待接真實系統

> 核心原則：**判定 / 狀態機 / 渲染邏輯維持不變**，只替換「資料從哪來」。

- **RFID 事件流（建議 WebSocket / SSE）**：進槽 / 離槽 / 在槽偵測（未偵測到 = 未放回 / 離槽），取代原型的手動拖曳與 `pulledFromTank` 判斷。
- **上位系統 (Host/MES)**：依 PN/SN 回傳 `{規格酸種, 要求時間}`（規則 4），取代原型的本地查表。找不到 PN 需有明確失敗狀態。
- **酸桶設定**：`{tankId, benchId, acid}` 來自 MES；換酸時更新並重判錯槽。
- **紀錄持久化（原型缺，需新增後端）**：離槽 Record 寫入資料庫供稽核 / 報表；最終 result 建議由後端在離槽事件時計算落庫，避免前端時鐘 / 離線造成不一致。

## 7. 驗收條件 (Acceptance Criteria)

1. 流程單向：不可回頭 / 跳站；Signin→IQC→Tank→HPW 正確擋關（規則 1）。
2. 一桶一酸：規格酸種 ≠ 本桶 → 錯槽禁止進槽（規則 2）。
3. 酸種配對：PW 槽需已泡過酸才可進（規則 3）。
4. 參數 per-PN：同桶不同零件可有不同要求時間（規則 4）。
5. 計時到秒、判定到分、1 分鐘寬限：達標→可取出綠；超過 1 分鐘→超時紅並記錄超時（規則 5）。
6. 異常：未達標被取出→異常、保留時間、放回可接續；未放回→現場螢幕「未偵測到 Tag・請放回」（規則 6）。
7. 進 HPW → 解除所有標記，流程結束。
