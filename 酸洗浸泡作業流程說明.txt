============================================================
 酸洗浸泡產線 — 作業流程說明（純流程 / UX，不含畫面操作）
 Acid Soak Line — Process & UX Overview
============================================================

【一句話總覽】
零件貼上 RFID 標籤後，依「簽入 → 進料檢驗 → 酸洗浸泡（依規格對應酸種）→
（必要時）超純水潤洗 → 最終潤洗結束」的單向流程逐站流動；系統全程以
RFID 追蹤每件零件的位置與浸泡時間，並在異常時即時警示現場人員。


------------------------------------------------------------
一、基本概念與名詞
------------------------------------------------------------
• Runcard（流程卡）：一件（批）零件的製程身分；貼有 RFID 標籤，系統靠它
  辨識「這是誰、走到哪、泡多久」。卡號如 RC-2026063xx。

• PN / SN：零件料號 / 序號，由 RFID 讀回。系統依 PN/SN 決定該零件的
  規格酸種與要求浸泡時間。

• 規格酸種（STM-14 / STM-31 / STM-07…）：這件零件「該泡哪一種酸」，
  是零件本身的規格屬性，固定跟著 Runcard。

• 一桶一酸：每個酸槽（Tank）只裝「一種」液體 —— 某一種酸，或超純水（PW）。
  酸種是「酸槽」的屬性，不是零件可任選的欄位。

• Soak Bench（浸泡槽台）與 Tank（酸槽）：一個 Bench 上有數個 Tank
  （如 Tank A / Tank B）。現場一個 Bench 對應一個顯示螢幕。


------------------------------------------------------------
二、單向作業流程（不可回頭、不可跳站）
------------------------------------------------------------
每件零件必須依序通過下列站別，中途不得倒退或略過任何一站：

  ① 簽入 Signin
  ② 進料檢驗 IQC
  ③ 酸洗浸泡 Tank（清洗區；可在同一 Bench/不同 Tank 間流動）
  ④ 最終潤洗 HPW（結束）

各站說明如下。

① 簽入 Signin
   零件進線的第一關，確認這件零件正式進入本製程、開始被系統追蹤。
   未簽入者不得進入後續任何站別。

② 進料檢驗 IQC
   對零件做進料檢驗；必須「已簽入」才能進行檢驗。
   完成 Signin + IQC 後，這件零件才具備「可進酸槽」的資格。

③ 酸洗浸泡 Tank（核心）
   系統會依排班，提示「該把哪一張 Runcard 放進哪一個酸槽」。
   進槽需同時滿足兩個條件：
     (a) 已完成 Signin + IQC；
     (b) 酸種相符 —— 若槽內是酸，零件的規格酸種必須與該槽酸種一致；
         若槽內是超純水（PW），零件必須「已先泡過酸」才可進入（見第四節）。
   不相符者一律不得進槽（防呆：錯槽）。
   進槽後即開始「浸泡計時」（見第三節）。

④ 最終潤洗 HPW
   完成所有浸泡/潤洗後，零件進入 HPW 作最終潤洗，流程即結束；
   此時系統解除該零件的所有製程標記（簽入、檢驗、泡酸紀錄、異常等）。


------------------------------------------------------------
三、浸泡計時與達標判定（判定到「分」）
------------------------------------------------------------
• 現場計時：零件在酸槽內，系統即時往上累計「已浸泡時間」。

• 要求時間：由零件的 PN/SN 決定（不同零件可有不同要求時間，即使在同一槽）。

• 達標 / 可取出：已浸泡時間達到要求時間，即為「達標，可取出」。

• 到「分」判定與 1 分鐘寬限：
  最終判定以「分」為單位認定。達標後給予 1 分鐘寬限期，這段期間仍視為
  「達標可取出」，不算超時 —— 避免因差幾秒/未滿一分鐘而被誤判 NG。

• 超時（過浸）：超過要求時間「1 分鐘以上」才判定為「超時」，提醒立即取出，
  並於離槽紀錄中記錄超時多久，以利追溯。

  判定結果彙整：
    - 達標 PASS：在要求時間 ~ 寬限內取出。
    - 超時 OVER：超過要求時間 1 分鐘以上取出。
    - 異常 NG ：見第五節。


------------------------------------------------------------
四、酸種配對與「超純水潤洗（PW）」前置條件
------------------------------------------------------------
• 酸槽（酸）：零件規格酸種必須與槽內酸種一致，否則不得進槽。

• 超純水槽（PW）：用於「泡完酸之後」的潤洗。
  規則：零件必須「已先在相符的酸槽完成浸泡（已泡過酸）」，才允許進入 PW 槽。
  未泡過酸者不得直接進 PW，確保潤洗發生在正確的製程順序之後。

• 換液 / 移槽：酸槽可更換所裝液體（倒掉、換倒別種）；酸槽本身也可在
  Bench 之間調度。但槽內有零件（作業中）時不得換液，避免影響進行中的浸泡。


------------------------------------------------------------
五、異常處理（RFID 追蹤與現場警示）
------------------------------------------------------------
系統以 RFID 全程追蹤零件是否在正確位置。下列情形會即時警示現場人員：

• 未達標即被取出（倒數中被拿走）：判定為「異常」。
  - 系統會保留該零件「已浸泡的時間」，日後放回任一酸槽時可「接續計時」，
    不需從頭重泡。
  - 異常標記跟著零件走，直到零件完成流程（進 HPW）或被重置。

• 未偵測到 Tag / 請放回：
  當一件零件在浸泡中被取出、卻尚未放回任何酸槽時，該酸槽對應的
  現場螢幕會顯示紅色警示「未偵測到 Tag・請放回零件」，提醒作業人員把
  零件放回槽內繼續作業。零件一旦放回（接續計時）或完成流程，警示即解除。

• 移出區（暫置）：被（異常）取出的零件可暫置於「移出區」，
  代表零件目前不在任何酸槽中、等待處置或放回。移出區僅供零件暫放。


------------------------------------------------------------
六、現場螢幕（作業人員視角）
------------------------------------------------------------
每個 Soak Bench 對應一個現場螢幕，一次顯示「該 Bench 內所有酸槽（如 Tank
A/B）」，讓作業人員一眼掌握整台狀態。每個酸槽區塊顯示：
  - 酸槽所裝的酸種 / PW（直接標示）；
  - 目前狀態：等待進入 / 請放入某張卡 / 作業中 / 達標可取出 / 超時 / 異常 /
    未偵測到 Tag 請放回；
  - 已浸泡時間與要求時間；
  - 槽內零件資訊：Runcard、PN、SN、規格酸種。
不同狀態以顏色區分（作業中=一般、可取出=綠、超時/異常=紅並閃爍）。


------------------------------------------------------------
七、監控站（管理 / 稽核視角）
------------------------------------------------------------
以「酸槽」為單位的即時監控牆：
  - 顯示各槽內零件的要求 / 已浸泡 / 進度 / 狀態；
  - 頂部橫幅彙總異常、以及「被取出未放回」的零件數；
  - 「出槽紀錄」逐筆記錄每次零件離槽的結果（達標 PASS / 超時 OVER /
    異常 NG）與時間，供事後追溯與稽核。


------------------------------------------------------------
八、一件零件的典型旅程（範例）
------------------------------------------------------------
  1. 貼 RFID → 簽入 Signin。
  2. 進料檢驗 IQC。
  3. 系統排班提示 → 放入「與其規格酸種相符」的酸槽，開始浸泡計時。
  4. 達標（並在 1 分鐘寬限內）→ 判 PASS，取出。
  5. （若製程需要）放入超純水 PW 槽潤洗（因已泡過酸，允許進入）。
  6. 最終潤洗 HPW → 流程結束，系統解除所有標記。

  ＊若步驟 3 未達標就被拿走 → 異常，時間保留，現場螢幕提示放回；
    放回後接續計時，最終仍可完成流程。
